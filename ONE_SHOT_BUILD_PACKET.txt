ICT MSS SESSION ENTRY MODEL v1

CANONICAL ONE-SHOT PINE V6 BUILD PACKET

0. Build Role

You are generating a Pine Script v6 indicator, not a strategy.

The output must implement a single-chain ICT session model with this strict progression:

ASIA_KZ build -> LONDON extend -> external freeze at 07:00 NY -> first NY sweep -> MSS -> PD filter -> filtered POI render

The script must remain sparse, clean, and deterministic.
If the chart looks busy before the full chain resolves, the build is wrong.

3. Execution-Day Ownership

This is mandatory.
	•	The execution day is the New York calendar day on which LONDON and NEW_YORK_KZ occur.
	•	The ASIA_KZ session for execution day D runs from:
	•	20:00 NY on D-1
	•	to 00:00 NY on D
	•	Therefore, the Asia range belongs to the following London/NY execution day.

This must be implemented correctly or the whole model binds the wrong Asia range to the wrong NY session.

4. Session Definitions

ASIA_KZ
	•	Time: 20:00 -> 00:00 NY
	•	Track High/Low: yes
	•	Draw Box: yes
	•	Purpose: build initial range

LONDON
	•	Time: 00:00 -> 05:00 NY
	•	Track High/Low: yes
	•	Draw Box: no
	•	Purpose: extend range

NEW_YORK_KZ
	•	Time: 07:00 -> 11:00 NY
	•	Track High/Low: no
	•	Draw Box: no
	•	Purpose: execution window only

Asia Carry Rule

Those locked Asia values belong to the new execution day.

Implementation rule:

London Range

10. External Liquidity Freeze

At the start of NEW_YORK_KZ, exactly 07:00 NY:
	•	externalHigh = max(asiaHigh, londonHigh)
	•	externalLow = min(asiaLow, londonLow)

Rules:
	•	compute once
	•	lock once
	•	never update afterward within the same execution day
	•	only one external high and one external low may exist

Visuals:
	•	draw one horizontal line at externalHigh
	•	draw one horizontal line at externalLow
	•	both extend forward until swept or new day reset

⸻

11. Sweep Engine

Valid Window

Sweep detection is valid only:
	•	inside NEW_YORK_KZ
	•	on confirmed bars
	•	before any prior valid sweep has been recorded

Sweep Definitions
	•	Sell-side sweep: high > externalHigh
	•	Buy-side sweep: low < externalLow

Dual-Sweep Rejection

If the same confirmed NY bar satisfies both:
	•	high > externalHigh
	•	low < externalLow

then reject that bar as a valid sweep bar.

No sweep is recorded on that bar.

First Sweep Law

Only the first valid sweep per execution day is stored.

Stored Sweep Fields

When a valid sweep occurs:
	•	sweepDirection
	•	sweepPrice
	•	sweepTime
	•	sweepHigh = high of sweep bar
	•	sweepLow = low of sweep bar

Line behavior:
	•	if externalHigh is swept, stop that line at the sweep bar
	•	if externalLow is swept, stop that line at the sweep bar

⸻

12. Premium / Discount Filter

Internal only. Never draw it.

Define:
	•	equilibrium = (sweepHigh + sweepLow) / 2

Rules:
	•	Bullish setup: only POIs with midpoint strictly below equilibrium are valid
	•	Bearish setup: only POIs with midpoint strictly above equilibrium are valid
	•	Equality fails

⸻

13. MSS Engine

This build does not reference any missing external MSS logic.
The MSS logic is fully local and defined here.

MSS Mode

Use a confirmed pivot-based MSS engine.

Pivot Length

Canonical default:
	•	pivot length = 2

This may be exposed as an input, but default must be 2.

Pivot Definitions
	•	Swing High = confirmed pivot high
	•	Swing Low = confirmed pivot low

Only confirmed pivots may be used.

MSS Eligibility

MSS can occur only if:
	•	inside NEW_YORK_KZ
	•	on confirmed bars
	•	after a valid sweep has already been recorded
	•	no prior MSS has been confirmed for the day

Bullish MSS

After a buy-side sweep:
	•	wait for the first confirmed swing high that exists after the sweep
	•	bullish MSS occurs on the first confirmed NY bar that closes above that swing high

Bearish MSS

After a sell-side sweep:
	•	wait for the first confirmed swing low that exists after the sweep
	•	bearish MSS occurs on the first confirmed NY bar that closes below that swing low

Stored MSS Fields

When MSS confirms:
	•	mssDirection
	•	mssLevel
	•	mssTime
	•	mssPivotTime

MSS Visual

Draw exactly one MSS line:
	•	from pivot time
	•	to break time
	•	at broken structure level

Visibility:
	•	NY only
	•	delete when NY ends

14. Displacement Leg Definition

The displacement leg causing MSS is:
	•	the bar sequence beginning immediately after the sweep bar
	•	ending at the MSS confirmation bar inclusive

This displacement leg is the only valid sourcing window for the Order Block.

⸻

15. Order Block Engine

Bullish OB

After bullish MSS:
	•	scan backward through the displacement leg only
	•	select the last bearish candle
	•	box bounds:
	•	top = candle high
	•	bottom = candle low

Bearish OB

After bearish MSS:
	•	scan backward through the displacement leg only
	•	select the last bullish candle
	•	box bounds:
	•	top = candle high
	•	bottom = candle low

OB Eligibility

OB may render only if:
	•	MSS is already confirmed
	•	OB exists inside displacement leg
	•	OB is inside NY
	•	OB passes PD filter

⸻

16. FVG Engine

Evaluate only on confirmed NY bars at or after MSS confirmation.

Bullish FVG

Exists when:
	•	low[1] > high[2]

Bounds:
	•	top = low[1]
	•	bottom = high[2]

Bearish FVG

Exists when:
	•	high[1] < low[2]

Bounds:
	•	top = low[2]
	•	bottom = high[1]

FVG Eligibility

Valid only if:
	•	created after MSS
	•	created inside NY
	•	passes PD filter

⸻

17. IFVG Engine

Bullish IFVG

A prior bearish FVG becomes bullish IFVG when:
	•	price trades through that bearish FVG
	•	and a confirmed NY bar closes above the top of that bearish FVG

Bearish IFVG

A prior bullish FVG becomes bearish IFVG when:
	•	price trades through that bullish FVG
	•	and a confirmed NY bar closes below the bottom of that bullish FVG

IFVG Eligibility

Valid only if:
	•	source FVG is valid
	•	inversion occurs after MSS
	•	inversion occurs inside NY
	•	passes PD filter

18. POI Render Policy

Maximum Visible POIs

Show at most:
	•	1 OB
	•	1 FVG
	•	1 IFVG

Maximum visible total = 3

Winner Rule

If multiple candidates of the same type qualify:
	•	keep only the most recent valid candidate
	•	replace the prior one

No Pre-MSS Rendering

No POI may render before MSS.

No Outside-NY Rendering

No POI may render outside NY.

If None Exist

Show nothing.
